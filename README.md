# vk_marketplace

## Описание проекта

Это REST API для маркетплейса объявлений, реализованный на Go. Основные возможности сервиса:

* Регистрация и авторизация пользователей с JWT-аутентификацией.
* Работа с куками для безопасного хранения токена.
* Создание объявлений (title, description, price, image).
* Получение ленты объявлений с курсорной пагинацией.
* Валидация входящих данных с помощью тегов и кастомных правил.
* Гибкая сортировка объявлений по дате создания или цене в любом порядке.

## Начало работы

### Конфигурация проекта

Проект использует два источника конфигурации:

#### 1. YAML-файл

```yaml
server:
  host: localhost               # Хост, на котором запускается сервер
  port: 8080                    # Порт, на котором запускается HTTP-сервер
  max_image_size_bytes: 5242880 # Максимальный размер изображения (в байтах) — 5 МБ
  feed_limit: 2                 # Количество объявлений по умолчанию в ленте
```

---

#### 2. Переменные окружения (`.env`)

```env
# Ключи шифрования и подписи

APP_JWT_SECRET_KEY=my_secret_key     # Секретный ключ для подписи JWT-токенов
APP_AES_SECRET_KEY=my_secret_keyyyy # Секретный ключ для симметричного AES-шифрования (16, 24, 32 байта)

# Настройки PostgreSQL

PG_HOST=localhost                    # Адрес сервера PostgreSQL
PG_PORT=5432                         # Порт подключения
PG_DATABASE_NAME=marketplace_db      # Название базы данных
PG_USER=admin                        # Имя пользователя PostgreSQL
PG_PASSWORD=1234                     # Пароль пользователя
PG_SSLMODE=disable                   # Режим SSL-соединения (disable — для локальной разработки)

# Миграции

MIGRATION_DIR=./migrations           # Путь до папки с SQL-миграциями
```

### Запуск проекта
Для запуска PostgreSQL и накатывания миграций выполните:

```sh
make pg-up
make migration-up
```

Для запуска проекта доступны несколько режимов, которые управляются флагами командной строки:

```sh
  -config-path your/congig.yaml    путь до YAML-конфига
  -local                           запуск в локальном режиме
  -validation                      включение валидации входных данных
```

#### Локальный запуск **с валидацией** (через Makefile)

```bash
make local-up
```

> Под капотом запускается:
> `go run cmd/main.go -config-path=config/local/config.yaml -local -validation`

## Архитектура и реализация

Проект построен по принципам **чистой архитектуры**, в которой слои логически и физически отделены друг от друга. Каждый уровень зависит только от уровней ниже, что обеспечивает тестируемость, масштабируемость и читаемость.

Основные принципы:

* **Модели и бизнес-логика** изолированы от фреймворков и библиотек.
* **Внешние интерфейсы (API, базы данных)** зависят от внутренней логики, но не наоборот.
* **Связи направлены внутрь** — только внутрь слоя (Dependency Rule).

### Основные директории

 `cmd/main.go` - корень запуска проекта

* Инициализирует конфигурацию.
* Создает логгер.
* Собирает зависимости: репозитории → сервисы → API-хендлеры.
* Запускает HTTP-сервер.
<br>
<br>

---

`internal/` - главный модуль приложения, делится на подкатегории по слоям архитектуры
<br>
<br>
`internal/api/` - cлой **обработчиков HTTP-запросов** — контроллеры:
* `advertisement/` — Create, Get, Update, Delete объявлений.
* `auth/` — SignUp, SignIn, работа с JWT.
* `feed/` — работа с лентой (пагинация, курсор).
* `middleware/` — авторизационные и логирующие миддлвары.
* `error-processor.go` — приведение ошибок к единому JSON-формату.
<br>
<br>
`internal/config/` - загружает YAML-файл и парсит его в структуру Go.
<br>
<br>
 `internal/models/` - cущности и DTO:
* `advertisement.go`, `user.go` — бизнес-модели.
* `request.go`, `response.go` — структуры для API.
* `custom-errors.go` — кастомные ошибки.
<br>
<br>
`internal/repository/` - слой **доступа к данным** (база данных):

* `advertisement/` — работа с объявлениями.
* `user/` — создание пользователей, верификация.
* `repository.go` — фабрики и интерфейсы репозиториев.
<br>
<br>
`internal/sevice/` - бизнес-логика приложения:

* `advertisement/` — логика по управлению объявлениями.
* `auth/` — JWT, верификация, регистрация.
* `feed/` — генерация ленты, работа с курсорами.
* `mocks/` — автогенерируемые моки для тестирования.
* `service.go` — агрегирующая фабрика для сборки всех сервисов.
<br>
<br>
`internal/validator/` - проверка входных данных:

* Валидация объявлений, пользователей.
* `image-checker.go` — проверка ссылок на изображения.
* `errors.go` — стандартизированные ошибки валидации.

### Основные маршруты API:

#### Регистрация и вход

**POST** `/sign_up`
**POST** `/sign_in`

**Пример запроса**:

```json
{
  "login": "login",
  "password": "password123@"
}
```

---

#### Создание объявления

**POST** `/advertisements/create`
(доступно только авторизованным пользователям)

**Пример запроса**:

```json
{
  "title": "title",
  "description": "description",
  "price": 123,
  "image_url": "https://vk.ru/image.jpg"
}
```

---

#### Получение ленты объявлений

**GET** `/feed`
Поддерживает курсорную пагинацию, сортировку и ограничение количества элементов.

**Параметры запроса**:

| Параметр         | Тип        | Описание                                            |
| ---------------- | ---------- | --------------------------------------------------- |
| `cursor`         | `int`      | ID последнего объявления с предыдущей страницы      |
| `sort_by`        | `string`   | `date` или `price`                                  |
| `order`          | `string`   | `asc` или `desc`                                    |
| `created_before` | `datetime` | фильтрация по дате создания (опционально)           |
| `limit`          | `int`      | ограничение количества объявлений (по умолчанию значение в конфиге) |
